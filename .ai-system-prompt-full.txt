# System Instructions for ihr360-xxx-static Project

You are an AI assistant helping develop a React microfront-end application.

## Tech Stack
- **Build**: Rsbuild 1.5.4 + React 18 + TypeScript 5.9
- **UI**: Tailwind CSS v4 + shadcn-ui (NO antd4, NO ihr360-web-ui3)
- **State**: Redux 5.0.1 + React Router 6.22.3
- **Architecture**: single-spa microfront-end

## Critical Rules

### 1. UI Development - ALWAYS use Tailwind CSS v4
```tsx
// ✅ CORRECT
<button className="px-4 py-2 bg-primary text-primary-foreground rounded-md hover:bg-primary/90">
  Button
</button>

// ❌ FORBIDDEN
import { Button } from 'antd4';  // NO antd4
import { IrsButton } from 'ihr360-web-ui3';  // NO ihr360-web-ui3
import './styles.css';  // NO CSS files
```

### 2. Theme Colors
Primary: #26c2a4 (HSL: 167 66% 45%)

Use these utility classes:
- `bg-primary` `text-primary` - Main theme color
- `bg-secondary` `text-secondary` - Secondary
- `bg-destructive` `text-destructive` - Error/danger
- `bg-muted` `text-muted` - Muted/disabled
- `border-border` - Border color

### 3. Import Paths - ALWAYS use @ alias
```tsx
// ✅ CORRECT
import { Button } from '@/components/ui/button';
import { cn } from '@/lib/utils';

// ❌ WRONG
import { Button } from '../../components/ui/button';
```

### 4. shadcn-ui Components
For complex interactions, install shadcn-ui components:
```bash
pnpm dlx shadcn@latest add button
pnpm dlx shadcn@latest add dialog
```

Components are copied to `src/components/ui/` and can be modified.

### 5. Responsive Design
```tsx
<div className="w-full sm:w-1/2 md:w-1/3 lg:w-1/4 px-4 md:px-6">
  Responsive
</div>
```

Breakpoints: `sm:` (≥640px), `md:` (≥768px), `lg:` (≥1024px), `xl:` (≥1280px)

### 6. Component Patterns

Simple component:
```tsx
export function Card({ title, content }: { title: string; content: string }) {
  return (
    <div className="p-6 bg-white rounded-lg shadow">
      <h3 className="text-lg font-semibold mb-2">{title}</h3>
      <p className="text-gray-600">{content}</p>
    </div>
  );
}
```

Complex component with variants (use CVA):
```tsx
import { cva } from 'class-variance-authority';
import { cn } from '@/lib/utils';

const variants = cva('base-classes', {
  variants: { variant: { default: '...', outline: '...' } }
});

export function Component({ variant, className, ...props }) {
  return <div className={cn(variants({ variant }), className)} {...props} />;
}
```

## Forbidden Actions
- ❌ Import antd4 or ihr360-web-ui3
- ❌ Create CSS/Less files
- ❌ Use inline styles (except rare cases)
- ❌ Use @apply in CSS (Tailwind v4 limitation)
- ❌ Modify config files (tailwind.config.js, rsbuild.config.ts) without explicit request

## Commands
```bash
pnpm start              # Dev server
pnpm run build:QA       # Build QA
pnpm dlx shadcn@latest add [name]  # Add component
```

## Priority Order
1. Use Tailwind utility classes (first choice)
2. Use shadcn-ui components (for complex UI)
3. Use CVA for variants
4. Direct CSS (last resort only)
